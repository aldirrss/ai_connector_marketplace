# Packaging — Desktop App (Phase 5)

The AI Connector Marketplace ships as a [Tauri](https://tauri.app) v2 desktop app
that bundles the Next.js frontend and launches the FastAPI backend on startup.

```
┌──────────────────────────────────────────────┐
│  Tauri window (WebView)                        │
│  ├─ serves frontend/out (static export)        │
│  └─ fetch → http://127.0.0.1:8000              │
│                    │                           │
│  Rust shell (src-tauri)                        │
│  ├─ spawns: python -m uvicorn backend.main:app │
│  ├─ system tray (Show / Quit)                  │
│  └─ kills the backend on exit                  │
└──────────────────────────────────────────────┘
```

The backend runs as a child process using the user's Python interpreter, with the
bundled `backend/` and `registry/` directories as resources. This keeps the binary
small and avoids freezing a Python runtime into the bundle.

## Layout

```
package.json            ← Tauri CLI + scripts (tauri:dev, tauri:build, icon)
src-tauri/
├── Cargo.toml          ← Rust crate manifest
├── build.rs            ← tauri-build hook
├── tauri.conf.json     ← app + bundle config (icons, resources, window)
├── capabilities/       ← webview permission set
├── icons/              ← app icons (PNG; .icns/.ico generated by `tauri icon`)
├── app-icon.png        ← 1024² source icon for `tauri icon`
└── src/
    ├── main.rs         ← thin entrypoint
    └── lib.rs          ← backend launcher + tray + lifecycle
```

## Prerequisites

1. **Rust** (stable) — https://rustup.rs
2. **Node.js 18+** and the repo's npm deps (`npm install` at the root and in `frontend/`).
3. **Tauri system dependencies** for your OS — see
   https://tauri.app/start/prerequisites/. On Linux this includes
   `webkit2gtk-4.1`, `libappindicator`, `librsvg`, etc.
4. **Python 3.11+** available on the target machine (the app launches the backend
   with it). End users need Python + the backend deps installed:
   ```bash
   pip install -r backend/requirements.txt
   ```
   Override the interpreter with the `AICM_PYTHON` env var (e.g. point it at a venv).

## Develop

```bash
npm install            # installs @tauri-apps/cli at the repo root
npm run tauri:dev      # builds frontend (dev), launches the window + backend
```

`tauri:dev` runs `npm --prefix frontend run dev` and points the window at
`http://localhost:3000` with hot reload.

## Generate icons

The repo ships solid-brand placeholder PNGs. To regenerate the full icon set
(including `.icns` for macOS and `.ico` for Windows) from a source image:

```bash
npm run icon                       # uses src-tauri/app-icon.png
# or: npm run tauri icon path/to/your-logo.png
```

## Build installers

```bash
npm run tauri:build
```

This runs `npm --prefix frontend run build:tauri` (static export to
`frontend/out`) and produces native bundles in `src-tauri/target/release/bundle/`:

| OS | Artifacts |
|----|-----------|
| macOS | `.app`, `.dmg` |
| Windows | `.msi` (WiX), `.exe` (NSIS) |
| Linux | `.AppImage`, `.deb`, `.rpm` |

> Bundles are **platform-specific** — build each target on its own OS (or in CI
> with a matrix). There is no cross-compilation for the WebView layer.

## Auto-update & code signing (TODO)

These require secrets/certificates that live with the maintainer, not in the repo:

- **Updater:** add the `tauri-plugin-updater`, host a signed `latest.json` +
  artifacts, and set `plugins.updater` in `tauri.conf.json` with the public key.
- **macOS:** Developer ID certificate + notarization (`APPLE_*` env vars).
- **Windows:** Authenticode certificate (`signtool` / Azure Trusted Signing).

A CI matrix (macos / windows / ubuntu) calling `tauri-apps/tauri-action` is the
recommended way to produce signed installers for a release.

## Notes / current limitations

- The desktop window's WebView origin is not `localhost:3000`; the backend CORS
  list already includes the Tauri origins (`tauri://localhost`,
  `https://tauri.localhost`, `http://tauri.localhost`).
- The shipped icons are simple placeholders — replace `src-tauri/app-icon.png`
  and run `npm run icon` before a real release.
- Building the GUI binary was **not** verified in the development sandbox (no
  WebKit/system deps there); the web export, configs, and Rust source are in
  place and should build once the prerequisites above are installed.
